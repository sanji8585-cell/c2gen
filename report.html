2. 실제로 없는 기능 - UX 개선 제안

A. 플레이그라운드 소셜 기능 부족

현재 상태: 좋아요만 있음 (Playground.tsx:127-158). 댓글, 팔로우, 북마크, 태그 없음.

없는 기능영향
댓글 시스템커뮤니티 상호작용 없음 → 좋아요만으로는 재방문 동기 부족
팔로우/팔로워좋은 크리에이터를 찾아도 계속 볼 방법 없음
북마크/컬렉션마음에 드는 작품 저장 불가
태그/해시태그검색이 제목/작성자명으로만 가능. 주제별 탐색 불가
조회수얼마나 인기 있는지 좋아요 외에 판단 기준 없음

B. 프리셋 확장 (구현 완료)

해결됨: 기존에 이미지 모델, TTS 음성, TTS 모델 3가지만 저장되던 프리셋을 화풍(Gemini/GPT), TTS 속도/안정성, 언어, 영상 방향, 한글 억제까지 포함하도록 확장. 서버(Supabase) 저장으로 전환하여 어디서든 동기화 가능.

C. 구독 기능 미완성

현재 상태: CreditShop.tsx:311에서 구독 버튼 클릭 시 "구독 기능은 준비 중입니다" 메시지만 표시. 플랜 탭이 그대로 노출됨.

  • 구독 준비 전이면 플랜 탭에 "Coming Soon" 뱃지를 붙이거나, 팩 구매 유도 문구로 대체
  • 연간 플랜 추가 (월간 대비 15-20% 할인) - SaaS 기본 패턴

D. 크레딧 부족 시 생성 중단 위험

현재 상태: App.tsx:648에서 최소 10크레딧만 확인하고 시작. 10씬 기준 예상 비용은 200+ 크레딧이지만, 중간에 부족해도 일부만 생성되고 끝남.

스크립트 리뷰 단계에서 비용 예측 카드(App.tsx:1724)는 잘 되어 있으나, 예상 비용 > 잔액일 때 "생성 시작" 버튼을 비활성화하거나 경고하는 로직이 없음.

제안: handleApproveScript 시점에서 예상 비용 vs 잔액 비교 → 부족하면 버튼 색상 변경 + "크레딧 부족 (예상 320 / 잔액 50)" 표시 + 충전 유도

4. 게이미피케이션 고도화

A. 가챠 연출 부재

현재: InventoryModal의 가챠 탭에서 버튼 클릭 → pullGacha() → 결과가 GameOverlay로 바로 표시. 뽑기 과정의 연출 애니메이션이 없음.

  • 카드 뒤집기 or 슬롯머신 회전 애니메이션 (1-2초)
  • 레전더리 등급 시 화면 흔들림 + 금빛 이펙트
  • 10연차 뽑기 옵션 (레어 이상 1개 보장)

B. 시즌 이벤트 시스템

현재: EventBanner.tsx와 관리자 EventManager.tsx 인프라는 있으나 실제 시즌 콘텐츠(한정 아이템, 시즌 미션)는 관리자가 수동 생성 필요.

  • 시즌 전용 가챠 풀 (한정 프레임/칭호)
  • 시즌 미션 보드 (예: "이번 달 10개 영상 만들기")
  • 시즌 종료 시 한정 뱃지 자동 지급

C. 창작 챌린지

현재: 없음. 플레이그라운드에 작품은 올라오지만 주제별/챌린지별 모아볼 수 없음.

  • 주간/월간 챌린지 주제 (관리자 설정)
  • 참여 작품에 챌린지 태그 자동 부착
  • 투표 → 상위 3명 크레딧 보상

6. 기술적 주의점

A. 메모리 - base64 이미지 in state

현재: assetsRef.current에 씬별 base64 이미지 원본 저장. 30씬 x ~1-2MB = 30-60MB가 브라우저 메모리에 상주.

Undo 스냅샷(useUndoRedo.ts)이 얕은 복제라 base64 문자열은 참조 공유하므로 undo 자체는 메모리 부담 적음 (잘 설계됨). 하지만 씬 수가 많을수록(40-50개) 느려질 수 있음.

제안: 장기적으로 IndexedDB에 이미지 캐싱, state에는 참조 ID만 보관

B. 4K 렌더링

현재: config.ts:355-362에서 3840x2160, 35Mbps 비트레이트. Canvas 기반 브라우저 렌더링이므로 모바일이나 저사양 PC에서 메모리 초과 위험.

제안: 4K 선택 시 경고 문구 강화 ("PC에서만 권장" 정도를 더 명확하게)
